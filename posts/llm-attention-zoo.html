LLM Attention 变体 Zoo: 可点击的家族树

引子

现代 LLM 的 attention 沿几个相对独立的轴在演化:KV head 结构(MHA / MQA / GQA / MLA / CCA)、计算形式(dense softmax / sparse / linear / SSM / mLSTM)、稀疏模式(SWA / NSA / DSA / MoBA)、混合架构(KDA+MLA、Lightning+MLA、Mamba+Attn)、正交件(QK-Norm、Partial RoPE、NoPE、Gated Attention)。 静态表格能塞下信息但读起来累,纯描述容易丢全局。

核对范围:本文最后核对日期是 2026-06-04。MHA / MQA / GQA / MLA、FlashAttention、Mamba、RWKV、xLSTM 这类成熟分支按公开论文和主流实现整理;2025-2026 的新模型叶节点按公开 model card、技术报告或项目页整理。带 (?) 的仓库或模型归类只是待核对线索,不应当当成已确认事实引用。

这篇把它们放进一棵 可点击展开的家族树:

  • 第一层按 attention 计算形式分大类。
  • 第二层按 KV head 结构或子家族。
  • 叶节点是具体模型/论文。
  • 点击节点展开/折叠它的子树,hover 显示一句话定义。

怎么读这棵树

  • 圆点是蓝的:这个节点有可展开的子节点(branch)。点一下展开 / 收起。
  • 圆点是灰的:叶节点(具体模型)。
  • hover 任意节点:鼠标旁弹出一句话定义。
  • 右上角两个按钮可以一键全部展开或仅留第 1 层。
  • 所有节点都是同一个语义层。 Hybrid 混合架构归到它占主导的那一支(比如 Kimi Linear 走 linear,即使 25% 的层是 MLA 全注意)。

主树:Attention 形式 × KV head × 模型

第一层是 attention 的计算形式(决定复杂度数量级),第二层是 KV head 结构或子家族,叶节点是具体模型。

LLM Attention Variant Zoo: A Clickable Family Tree

Intro

Modern LLM attention evolves along several mostly independent axes: KV-head structure (MHA / MQA / GQA / MLA / CCA), computation form (dense softmax / sparse / linear / SSM / mLSTM), sparsity pattern (SWA / NSA / DSA / MoBA), hybrid architectures (KDA+MLA, Lightning+MLA, Mamba+Attn), and orthogonal building blocks (QK-Norm, Partial RoPE, NoPE, Gated Attention). A static table fits the information but reads dryly, and a pure narrative loses the global shape.

Scope: last checked on 2026-06-04. Established branches such as MHA / MQA / GQA / MLA, FlashAttention, Mamba, RWKV, and xLSTM are organized from papers and common implementations. Newer 2025-2026 model leaves are based on public model cards, technical reports, or project pages. A (?) marker means the repo or placement is still a lead to verify, not a confirmed citation.

This article puts them into a clickable family tree:

  • Top level by computation form.
  • Middle level by KV-head structure or sub-family.
  • Leaves are specific models / papers.
  • Click a node to expand or collapse its subtree; hover for a one-line definition.

How to read it

  • Blue dot: branch node with children. Click to expand or collapse.
  • Grey dot: leaf node (a specific model).
  • Hover any node: a tooltip appears with a one-line definition.
  • Top-right buttons toggle expand-all or collapse to depth 1.
  • Every node sits in exactly one place. Hybrid architectures are filed under the dominant branch (for example Kimi Linear sits under linear attention even though 25% of its layers are full MLA).

Main tree: attention form × KV head × models

Top level is the computation form (which sets the asymptotic compute and memory class); middle level is the KV-head structure or sub-family; leaves are concrete models.

辅助树:正交维度

下面这些维度跟主树正交:同一个模型可以在主树里走 GQA + dense softmax,同时在这棵辅助树里挂上 QK-Norm + Partial RoPE + Gated Attention。

Auxiliary tree: orthogonal dimensions

These dimensions are orthogonal to the main tree: a single model can sit under GQA + dense softmax in the main tree while also carrying QK-Norm + Partial RoPE + Gated Attention in this auxiliary tree.

几点注释

  1. Hybrid 模型的归属:层间混合(layer-interleave)的模型按"占多数的那一支"归类。比如 Kimi Linear 的 KDA:MLA = 3:1,主支是 KDA,所以挂在 linear 这一支;它在 MLA 那一支也会被提到,但只作为引用。
  2. "no KV / stateful":这一支放弃了显式 KV cache,改用一个固定大小的 state(矩阵或向量)替代。它在树里跟 MHA / GQA / MLA 同层,但所在的概念空间不一样,既不是 KV-head 结构的延伸,也不是稀疏化路线的延伸。
  3. DeepSeek V4 家族(CCA / CSA / HCA / mHC):挂在 sparse 一支下,但实际上跨多个轴(KV 压缩 + sparse + hyper-connection)。 2026 年新东西,公开材料有限,树里只放占位,具体语义建议直接读 V4 技术报告。
  4. 把每一支单独写成深度笔记是这个系列接下来要做的事:KV head 演进(MHA → MLA)、Linear attention 家族数学推导、Sparse attention(NSA / MoBA / DSA)的硬件对齐、SSM 与 attention 的等价性。

A few notes

  1. Where hybrids live: layer-interleave models are placed under the dominant branch. Kimi Linear's KDA:MLA = 3:1 puts it under linear attention; it also appears as a cross-reference under MLA but only as a pointer.
  2. The "no KV / stateful" branch: this family gives up the explicit KV cache entirely and stores a fixed-size state (matrix or vector) instead. It sits at the same tree level as MHA / GQA / MLA but in a different conceptual space, separate from both the KV-head axis and the sparsity axis.
  3. The DeepSeek V4 cluster (CCA / CSA / HCA / mHC) is filed under sparse but actually spans multiple axes (KV compression, sparsity, hyper-connections). These are 2026 entries with limited public detail; the tree carries placeholders, and the V4 technical report is the authoritative reference.
  4. Each major branch deserves its own deep-dive post, which is the plan for the rest of this series: KV-head evolution (MHA → MLA), the math of the linear-attention family, sparse-attention hardware alignment (NSA / MoBA / DSA), the equivalence between SSM and linear attention.
